# Ozon Tech E-Cup 2025 ML Challenge

> **Конкурсное решение задачи машинного обучения, разработанное в рамках Ozon Tech E-Cup 2025. Проект охватывает полный цикл построения ML-модели: от исследования данных и генерации признаков до обучения, валидации и подготовки финального решения для соревнования.**

---

# 📖 О проекте

**Ozon Tech E-Cup 2025 ML Challenge** — проект, посвящённый решению прикладной задачи машинного обучения на реальных данных в рамках соревнования **Ozon Tech E-Cup 2025**.

Основной целью проекта является построение модели, способной максимально точно прогнозировать целевую переменную на основе большого количества признаков. Работа включает все ключевые этапы современного ML-пайплайна: исследовательский анализ данных, очистку и подготовку датасета, построение новых признаков, обучение моделей, подбор гиперпараметров и генерацию итогового файла для отправки на платформу соревнования.

Проект ориентирован не только на получение высокого качества предсказаний, но и на создание воспроизводимого и удобного процесса разработки моделей машинного обучения.

# ⚙️ Pipeline решения

## 1. Исследование данных

На первом этапе проводится исследовательский анализ данных:

- анализ распределений признаков;
- исследование пропущенных значений;
- поиск выбросов;
- изучение взаимосвязей между признаками;
- оценка информативности признаков.

---

## 2. Подготовка данных

Перед обучением модели выполняются следующие этапы предобработки:

- очистка данных;
- обработка пропусков;
- кодирование категориальных признаков;
- масштабирование числовых признаков (при необходимости);
- формирование обучающей и валидационной выборок.

---

## 3. Feature Engineering

Одним из ключевых этапов проекта является построение новых признаков.

Используются:

- статистические признаки;
- взаимодействия между признаками;
- агрегированные характеристики;
- различные преобразования исходных данных.

Именно качественное пространство признаков зачастую обеспечивает основной прирост качества модели.

---

## 4. Обучение модели

После подготовки данных производится обучение модели машинного обучения.

Во время разработки выполняются:

- подбор гиперпараметров;
- сравнение нескольких моделей;
- кросс-валидация;
- анализ качества полученных результатов.

---

## 5. Генерация Submission

После выбора лучшей модели выполняется прогнозирование на тестовой выборке.

Полученные результаты автоматически сохраняются в формате, требуемом организаторами соревнования.

---

# 📈 Оценка качества

Качество модели оценивается с использованием официальной метрики соревнования.

Особое внимание уделяется:

- воспроизводимости результатов.
